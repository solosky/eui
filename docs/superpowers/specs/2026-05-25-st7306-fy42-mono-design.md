# ST7306 / FY42-Mono 显示驱动 — 设计文档

**日期：** 2026-05-25  
**状态：** 已批准  
**范围：** 为 FY42-Mono (YDP420H001-V3) 添加 ST7306 显示驱动，300×400 2bpp，同时添加 ESP-IDF CMake 集成。

## 1. 背景

FY42-Mono 是一款 4.2 寸、300×400 分辨率的反射式 TFT 显示屏，采用 ST7306 控制器（SPI 接口）。项目已具备 2bpp Canvas/Font 支持（提交 `8b9ed1e`–`5227b5b`）。参考实现位于项目根目录的 `st7306.h`（Retro-Go 风格，FreeRTOS SPI 任务架构）。本文档将该参考实现适配为符合项目 HAL 抽象模式的 EUI 显示驱动。

## 2. 驱动架构

### 文件

| 文件 | 用途 |
|------|------|
| `include/eui/driver/eui_drv_st7306.h` | 公开头文件：配置结构体 + create/destroy 原型 |
| `src/driver/eui_drv_st7306.c` | 实现：SPI 传输、像素打包、初始化序列 |

### 数据结构

```c
// 公开配置（用户传入）
typedef struct {
    eui_hal_spi_t spi;
    uint16_t      width;   // 300
    uint16_t      height;  // 400
} eui_drv_st7306_config_t;

// 驱动内部状态（.c 私有）
typedef struct {
    eui_display_drv_t base;
    eui_hal_spi_t     spi;
    uint16_t          width;
    uint16_t          height;
    uint8_t          *framebuffer;  // 150×200 = 30000 字节
} st7306_t;
```

### 继承

遵循 EUI 标准模式：`st7306_t.base` 作为第一个成员。无需 `container_of`——`eui_display_drv_t` 的 `user_data` 字段指向 `st7306_t`，所有虚表回调以 `void *ud` 形式接收 `user_data`。

### 缓冲模式

`EUI_BUFFER_FULL`——通过 `eui_malloc()` 分配 30KB 原生格式帧缓冲区。缓冲区与 ST7306 GRAM 布局完全一致：每行 150 字节 × 200 行，每像素 2bpp。

### 像素打包（ST7306 原生格式）

每字节代表一个 2×2 物理像素块（共 4 像素），每像素 2 bit：

```
Bit:  [7] [6] [5] [4] [3] [2] [1] [0]
      P0H P1H P0L P1L P2H P3H P2L P3L

  P0 = 左上像素 (bit1=7, bit0=5)
  P1 = 左下像素 (bit1=6, bit0=4)
  P2 = 右上像素 (bit1=3, bit0=1)
  P3 = 右下像素 (bit1=2, bit0=0)
```

逻辑坐标 (x, y) → 缓冲区偏移：

- `bx = x / 2`, `by = y / 2`
- `byte_idx = by * 150 + bx`
- `sx = x % 2`, `sy = y % 2`（2×2 块内的子像素位置）
- bit1 位置 = `7 - (sx * 4 + sy)`
- bit0 位置 = `7 - (sx * 4 + 2 + sy)`

### CASET/RASET 写窗口（刷新时使用）

50 个列地址 (5–54)，每地址对应 3 个缓冲字节 = 6 个逻辑像素。  
200 个行地址 (0–199)，每地址 = 2 个逻辑行。

刷新矩形映射：

- `gram_col_start = rect.x / 6`
- `gram_col_end   = (rect.x + rect.w - 1) / 6`
- `gram_row_start = rect.y / 2`
- `gram_row_end   = (rect.y + rect.h - 1) / 2`
- CASET = `(5 + gram_col_start)` .. `(5 + gram_col_end)`
- RASET = `gram_row_start` .. `gram_row_end`

SPI 数据逐行发送：每行 = `(gram_width * 3)` 字节。

### 初始化序列

从参考 `st7306.h`（第 406–467 行）原样搬移：NVM Load、Booster Enable、VCOM/Gate Voltage、OSC/Frame Rate、Gate Timing、Source EQ、Panel Setup、Sleep Out、MADCTL=0x48、COLMOD=0x11、Gamma Mode=0x20（单色模式）、CASET 5–54、RASET 0–199、Display ON。

### 虚表方法

| 方法 | 实现 |
|--------|---------------|
| `init` | 完整硬件初始化序列（约 25 条命令） |
| `deinit` | DISPOFF → SLPIN → 释放帧缓冲区 |
| `draw_pixel` | 读-改-写帧缓冲区 |
| `write_buffer` | Canvas 像素 → 原生缓冲 → 通过 CASET/RASET+RAMWR 进行 SPI 刷新 |
| `set_power` | DISPON / DISPOFF |
| `set_invert` | INVON / INVOFF |
| `set_contrast` | 空操作（反射屏，无背光） |
| `fill_rect` | 空操作（委托给 Canvas） |

## 3. ESP-IDF 集成

### CMake 自动发现

根 `CMakeLists.txt` 中 ESP-IDF 路径已使用 `file(GLOB EUI_DRIVER_SRCS src/driver/*.c)`，仅排除 `eui_drv_raylib.c`。将 `eui_drv_st7306.c` 放入 `src/driver/` 即可——ESP-IDF 路径无需修改 CMake。

### 配置模板

`configs/esp32/st7306_spi_300x400_2bpp.cmake`：

```cmake
set(EUI_TARGET_PORT      "esp-idf")
set(EUI_COLOR_DEPTH      2)
set(EUI_DISPLAY_DRIVER   "st7306")
set(EUI_DISPLAY_WIDTH    300)
set(EUI_DISPLAY_HEIGHT   400)
set(EUI_BUFFER_MODE      "full")
set(EUI_MEM_POOL_SIZE    65536)
set(EUI_PIN_SPI_HOST     1)
set(EUI_PIN_MOSI         23)
set(EUI_PIN_SCLK         18)
set(EUI_PIN_CS           5)
set(EUI_PIN_DC           16)
set(EUI_PIN_RST          17)
set(EUI_INPUT_DRIVER     "buttons")
set(EUI_PIN_BTN_UP       34)
set(EUI_PIN_BTN_DOWN     35)
set(EUI_PIN_BTN_OK       32)
set(EUI_PIN_BTN_BACK     33)
```

### Kconfig

`configs/esp32/st7306_spi_300x400_2bpp.Kconfig`——SPI 引脚、显示尺寸的菜单项。结构与现有 `st7735_spi_240x240_16bpp.Kconfig` 保持一致。无 BL 引脚（反射屏）。

### SPI HAL 工厂函数

在 `port/esp-idf/eui_port_esp_idf.c` 中新增 `eui_port_esp_idf_spi_create()`（目前仅有 I2C 工厂函数）。使用 ESP-IDF `driver/spi_master.h` 实现 `eui_hal_spi_t`（write_cmd、write_data、read_data、set_dc、set_cs、set_rst、delay_ms）。

### Bootstrap 更新

`port/esp-idf/eui_port_bootstrap.c` 当前硬编码 SSD1306/I2C。更新为根据 `EUI_DISPLAY_DRIVER` 分支：

```c
#if defined(EUI_DISPLAY_DRIVER) && EUI_DISPLAY_DRIVER == "st7306"
    // SPI + ST7306 路径
#else
    // 现有 I2C + SSD1306 路径
#endif
```

## 4. 桌面 CMake

在 `src/CMakeLists.txt` 的显式源列表中添加 `driver/eui_drv_st7306.c`。

## 5. 内存预算

| 项目 | 大小 |
|------|------|
| 帧缓冲区 (150×200×1 字节) | 30,000 字节 |
| 驱动结构体 + 开销 | ~200 字节 |
| EUI 核心 + 控件库 | ~4,000 字节 |
| **推荐内存池大小** | **65,536 字节** |

帧缓冲区通过 `eui_malloc()` 从 TLSF 内存池分配。用户需配置 `EUI_MEM_POOL_SIZE >= 65536`。
